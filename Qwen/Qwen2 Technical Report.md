# 升级
*   **模型家族扩充**：提供5种规格（$0.5B, 1.5B, 7B, 57B, 72B$），首次引入**混合专家模型（MoE）** `Qwen2-57B-A14B`。
*   **预训练数据量**：从`Qwen1.5`的$3万亿$Tokens翻倍至$7万亿$Tokens，涵盖约$30$种语言。
*   **基础架构升级**：所有模型均引入**分组查询注意力（`GQA`）**，实现了推理加速和显存降低。
*   **上下文窗口**：从Qwen1的$8K$扩展到$128K$`tokens`。
#  模型
*   **`GQA`（分组查询注意力）**：这是`Qwen2`架构上最关键的升级。所有尺寸模型都采用了`GQA`。简单来说，GQA在保证模型质量的同时，能显著降低**推理时**的内存占用并提升速度。`Qwen2.5`系列的后续成功也验证了这一架构的优越性。
*   **`MoE`（混合专家模型）**：`Qwen2`首次引入了`MoE`架构，打造了`Qwen2-57B-A14B`。它总参数量为$570亿$，但在推理时仅激活其中的$140亿$参数。这就像一个大型公司，虽然员工众多，但处理每个任务时只会调用最相关的专家小组，从而在性能和计算成本之间取得巧妙的平衡。
*   **分词器（Tokenizer）优化**：`Qwen2`沿用了高效的`BPE`分词器，词表大小约为$151k$个`token`。其在代码和数字上的高效编码，对于后续的数学和编程能力至关重要。
*   **位置编码创新**：`Qwen2`采用了旋转位置编码（`RoPE`），并通过`YaRN`等扩展技术，实现了长达$128K$$`tokens`的上下文长度支持。
# 训练
## 预训练数据 
报告明确指出，`Qwen2`的预训练数据量从`Qwen1.5`的$3万亿$`tokens`大幅提升至$7万亿$`tokens`。这$7万亿$`token`s并非简单的规模堆砌，而是经过精心筛选和配比，涵盖了广泛的领域和语言，尤其强化了代码和数学数据的比例。
## 后训练与对齐
`Qwen2-Chat`系列模型通过多阶段的对齐技术（SFT和RLHF），在多个维度上实现了全面提升：
*   **指令遵循**：能更准确地理解并执行用户的复杂指令。
*   **人类偏好对齐**：模型输出更符合人类对“有帮助、诚实、无害”的期望。
*   **安全对齐**：能更好地识别并拒绝对不安全或有害的查询，包括处理多种语言中的非法活动查询。
*   **多语言能力增强**：特别优化了多语言场景下的“语言转换”（code switch）问题，降低了模型在回答中意外切换语言的概率。
