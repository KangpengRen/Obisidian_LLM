# 核心升级
这次升级的核心，是坚持了“**数据驱动，全面进化**”的理念，并在规模上做到了极致。
*   **预训练：数据量的“疯狂”增长**：这是`Qwen2.5`性能飞跃的根本。它的预训练数据量从`Qwen2`的$7万亿 (7T)$ `tokens`大幅扩展到了惊人的$18万亿 (18T)$` tokens`。报告中的图表清晰地显示了随着数据量从$3T$到$18T$的迭代，模型在关键能力上的提升。除了规模，数据质量的控制也更加精细，尤其是大幅增加了数学和代码等高逻辑性数据的比例。
*   **后训练：更精细的“打磨”**：在监督微调(`SFT`)阶段，`Qwen2.5`使用了超过**100万个样本**，显著增强了模型的指令遵循、长文本生成和结构化数据分析能力。在强化学习(RL)阶段，它更是采用了创新的**两阶段RL**：先用**离线RL**在数学、编程等有标准答案的领域打好基础，再用**在线RL**根据真实性、无害性等人类偏好进行精细化调优。
##  1. 离线RL（Offline RL）
- **怎么做**：先收集好一大堆“状态-动作-奖励”的数据（比如数学题的正确解答、代码的运行结果），然后模型像做作业一样，反复从这批固定数据里学习，找到最优策略。
- **优点**：安全、稳定，不需要和环境实时交互，适合有标准答案的任务（比如数学计算、代码执行）。
- **在Qwen2.5中的应用**：报告提到，先使用离线RL在**数学、编程**等有明确对错的任务上打好基础。因为这类任务的反馈可以提前准备好（例如一道数学题的最终答案是否正确，一段代码能否通过测试）。
## 2. 在线RL（Online RL）
- **怎么做**：模型一边生成答案，一边接收实时反馈（比如人类偏好打分、安全性评估），然后立刻调整自己的行为。它是一个“行动→观察→改进”的循环。
- **优点**：能适应动态、复杂的偏好（比如“这个回答更礼貌”、“那个回答更安全”），这些偏好很难用固定数据提前标注完整。
- **在Qwen2.5中的应用**：在离线RL打好基础后，再使用在线RL针对**真实性、无害性、人类偏好**等主观指标进行精细化调优。例如，让模型实时生成多个回答，然后由奖励模型（或人工反馈）打分，模型根据分数即时更新。
# 总结
`Qwen2.5`技术报告清晰地展示了一条核心升级路径：**通过极致的数据规模扩展和精细的后训练打磨，将一个成熟的架构（`Qwen2`）推向了性能的新高度。** 它的成功也证明了，在算力允许的情况下，“数据规模”依然是大模型性能提升的强劲动力。
