# 模型
*   **通用架构设计**：所有模型均基于`Transformer`的**解码器（Decoder-Only）** 架构，并采用了多项主流先进技术以提升性能：
    *   **激活函数 (`SwiGLU`)**：提升了模型的非线性表达能力，使其能更好地学习复杂模式。
    *   **位置编码 (`RoPE`)**：使用旋转位置编码（Rotary Position Embedding），能更有效地处理长文本序列。
    *   **高效分词 (`BPE`)**：使用字节对编码（Byte-Pair Encoding）构建了约**15万**的超大词表。相比许多主流模型（词表通常为3-5万），`Qwen`的**超大词表**对中文和多语言的支持更高效。
*   **上下文长度（Context Length）**：报告详细列出了不同规格基础模型在预训练阶段能达到的上下文长度。
    *   **1.8B, 7B, 14B模型**：均支持$8192$个tokens。
    *   **72B模型**：支持$32768$个tokens。
*   **训练与对齐方法**：模型的后训练采用了多阶段技术。
    *   **监督微调（SFT）**：使用高质量的对话数据对模型进行初步指令微调。
    *   **人类反馈强化学习（RLHF）**：通过收集人类偏好数据来训练奖励模型，并利用强化学习算法（如PPO）对模型进行最终优化，使其行为更符合人类期望。
# 训练
## 预训练 (Pre-training)
这个阶段的目标是让模型从海量数据中学习语言结构、世界知识和基础推理能力，是构建通用知识的“通识教育”过程。具体做法如下：
*   **海量数据学习**：使用数万亿级别的高质量token进行训练。例如，`Qwen-72B`模型的预训练数据量就超过了**3万亿tokens**，而后续的`Qwen2.5-72B`更是扩展到了**18万亿tokens**。这些数据涵盖了网页、书籍、代码、百科等多样化的内容。
*   **高效模型架构**：`Qwen`采用了基于`Transformer`的优化架构，包含多个主流技术：
    *   **注意力机制**：采用分组查询注意力（`GQA`）和`Flash Attention`技术，在保证模型性能的同时，显著提升了训练和推理速度。
    *   **位置编码**：使用旋转位置编码（`RoPE`），让模型能更好地理解和处理长文本的序列顺序。
    *   **激活函数**：采用`SwiGLU`激活函数，增强了模型的非线性表达能力，使其能学习更复杂的模式。
    *   **优化器**：通常采用`AdamW`优化器来更新模型参数，并结合学习率预热（warmup）等策略，确保训练过程的稳定和高效。
*   **长文本能力**：通过分阶段的训练策略，模型能够处理的上下文窗口长度得到了显著提升。例如，`Qwen-72B`模型的上下文窗口在训练中最终达到了**32,768个token**。
## 后训练 (Post-training)
*   **有监督微调 (Supervised Fine-Tuning, SFT)**：
    模型会学习一个由海量高质量的“**指令-回答**”对构成的数据集。这些数据由人类专家编写或通过自动化流程生成，覆盖了对话、写作、翻译、代码生成等多种任务。例如，`Qwen2.5`模型的后训练阶段使用了超过**100万个SFT样本**。通过这个阶段的训练，模型学会了理解用户的指令并给出恰当的回应。
*   **人类反馈强化学习 (Reinforcement Learning from Human Feedback, RLHF)**：
    这是让模型行为与人类价值观对齐的关键一步。它让模型在微调过程中不断自我优化，使生成结果更安全、更有帮助。
    1.  **收集偏好数据**：人类标注员会对模型针对同一问题的多个回答进行排序，选出更好的那个。
    2.  **训练奖励模型 (Reward Model, RM)**：用这些偏好数据训练一个“裁判”模型，这个裁判能学会自动给模型的回答打分，分数高低代表回答质量的好坏。
    3.  **强化学习优化**：利用裁判的评分作为反馈，通过强化学习算法（如PPO）来持续优化`Qwen`模型的策略，让它越来越倾向于生成高分（也就是更符合人类偏好）的回答。
# 总结
基础模型`Qwen`和对话模型`Qwen-Chat`、代码模型、数学模型分离，`Qwen`即传统方式预训练出的模型，仅仅能预测下一词，而`Qwen-Chat`、代码、数学模型则是在`Qwen`的基础上进行有监督微调、人类反馈强化学习，使其成为更符合人类助手的工具。
