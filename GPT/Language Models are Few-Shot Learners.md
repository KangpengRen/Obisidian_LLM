# 背景
此前NLP领域的主流范式是“预训练+特定任务微调”，这种方法需要针对每个下游任务收集数千到数万条标注数据来微调模型，这种范式存在显著局限：每个任务单独准备大规模标注数据，难以覆盖海量潜在任务；模型易过拟合任务特定分布，泛化到新场景时性能下滑；与人类学习模式脱节，人类仅需要少量示例或自然语言指令就能掌握新任务。
`GPT-3`旨在验证一种全新的、更接近人类学习方式的范式：模型能否仅通过文本交互，无需微调，就能理解和完成新任务。
# 模型
`GPT-3`沿用了`GPT-2`相似的`Transformer decoder`架构，关键改进在于，其注意力层中使用了稀疏注意力模式（**Sparse Attention**），交替使用密集和局部带状稀疏注意力，降低了模型的计算复杂度，使得训练超大规模的模型成为可能（其最大版本拥有$1750M$参数，是`GPT`（$15M$）的$100$倍之多）。
模型在一个巨大的、经过精心筛选的混合数据集上进行训练，总计约$5000$亿个`token`，数据来源包括`Common Crawl`、`WebText2`、`Books1/2`和`Wikipedia`等，总规模达到$45TB$。训练规模也是“大力出奇迹”的体现，`GPT-3`是当时最大的稠密语言模型，耗费了数千`Petaflop/s-days`的算力。
**上下文学习（In-Context Learning）：**
`Gpt-3`核心发现是证明了“上下文学习”能力的存在，论文将其定义为一种“元学习”过程：
- **预训练阶段（外循环）**：模型在5000亿词元的海量数据上，学会了“根据前文模式续写后文”的通用能力。
- **推理阶段（内循环）**：当你给出“问题：... 答案：...”的示例时，模型识别出这是一个“问答模式”，并立即在内部模拟出“我现在要扮演一个问答机器人”的状态。

| 对比维度     | 微调                   | 上下文学习                    |
| -------- | -------------------- | ------------------------ |
| **参数变化** | **会变**。模型权重被永久修改。    | **不变**。模型参数冻结，处于推理模式。    |
| **学习信号** | **反向传播的梯度**。         | **注意力机制中的上下文**。          |
| **存储位置** | 知识写入模型的**长期记忆**（权重）。 | 知识暂存在**短期记忆**（KV Cache）。 |
| **遗忘风险** | 学新任务可能忘旧任务（灾难性遗忘）。   | 关掉对话框，新任务的影响就消失了。        |
| **数据需求** | 数千条标注数据。             | 0 到几十条自然语言示例。            |
# 总结
`GPT-3`通过一场大胆的“缩放实验”，实证了大模型“上下文学习”这一全新范式，标志着NLP领域从“专用模型”向“通用模型”的关键转折。尽管GPT-3取得了巨大成功，也伴有“幻觉”问题，它难以判断自己“知道”什么，同`GPT-2`一样，知识渊博但信口开河。
