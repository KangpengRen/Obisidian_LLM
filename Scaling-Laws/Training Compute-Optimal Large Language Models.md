# 背景

大模型训练一直遵循`OpenAI`的缩放定律：模型大小增长快，数据增长慢。结果是：`GPT-3`、`Gopher`等模型都只训练了约**3000亿token**，模型却越做越大。
# `Chinchilla`
这篇论文训练了**超过400个**不同规模的模型（7000万到160亿参数），用不同数据量（50亿到5000亿token）。
用**三种独立方法**分析数据，发现：
- 旧规律：模型增5.5倍，数据增1.8倍
- 新规律：模型增1倍，数据也增1倍
**验证：`Chinchilla`模型**
- 按照新规律，用**700亿参数 + 1.4万亿token**训练了一个模型，取名`Chinchilla`。
- 同样计算预算下，它比**2800亿参数的Gopher**表现更好：
  - MMLU准确率：67.6% vs 60.0%（提升7.6%）
  - 其他几十项任务全面领先
## 总结
| 之前以为                   | 现在发现                   |
| ---------------------- | ---------------------- |
| 模型越大越好                 | 模型和数据要一起涨              |
| 堆参数就完事了                | 堆数据同样重要                |
| GPT-3那种训练方式是合理的        | 那些大模型其实都“欠训练”了         |
一句话收尾：**与其花同样的钱做一个超大模型，不如做一个中等模型然后喂它海量数据——效果更好，推理更便宜。**
