# 背景
`OpenAI`发布`GPT-3`并同时提出了缩放定律（2020），通过训练从小到大的模型得到一个经验公式，即：
$$
性能=f(参数量,数据量,算力)
$$
核心结论为：在计算预算（算力）固定的情况下，模型**参数的增长权重 $>$ 数据量的增长权重**，也就是说：同等算力下，把数据做大不如把参数做大对性能提升更加明显，直接导致了“参数竞赛”，于是看到`GPT-3(175B)`、`Megatron(530B)`、`PaLM(540B)`这些模型的参数量越来越大，同时行业内认为“大就是好”！
而在大家都在严格遵守`OpenAI`的缩放定律堆参数量时，出现了一下问题：
1. **收益递减**：参数量从$13B$提升到$175B$的过程中性能提升巨大，而从$175B$到$540B$的过程中性能提升非常微小，但算力成本增长了$3$倍以上。原因为`OpenAI`的缩放定律基于小规模实验（几十亿参数）拟合出的曲线，它假设这种线性关系会一直延续。然而，模型越大，对数据量的需求是指数级增长的，数据量不够时，大模型就会“欠拟合”。
2. **数据瓶颈**：互联网上高质量、干净的文本数据是有限的（当时估计约为$5-10T$ `tokens`），当模型参数超过$1000$亿时，根据`OpenAI`原始公式，需要的训练数据量超过了互联网的综合。即模型太大了，数据喂不饱它。
之前的模型错误遵守了“缩放系数”，把参数量堆得太大，而数据喂的太少。
`DeepMind`发现了问题所在，他们重新做了实验，拟合了一条新的缩放定律曲线（`Chinchilla`法则，2022）：
- 旧定律（`OpenAI`）：参数量$N$优先级 $>$ 数据量$D$。
- 新定律（`DeepMind`）：参数量$N$和数据量$D$必须等比例缩放。算力增加$10$倍，$N$和$D$都应该增加$3.16$倍（$\sqrt 10$）。
# `LLaMA`
`LLaMA`的结构没有提出任何革命性的新算子，选择：
- `Pre-REMSNorm`：来自`GPT-3/LayerNorm`的变种；
- `SwiGLU`：激活函数，来自`PaLM`；
- `RoPE`：位置编码，来自`GPT-NeoX`等；
- `Multi-Query Attention`：仅用$65B$版本，来自以前的论文。
架构全是“抄作业”，把已经被验证有效的最佳实践集成在一起，`Meta`的贡献在于：确认这些组件组合在一起是稳定的、高效的，而非发明他们。
*Question：使用原始Transformer（Post-Norm + ReLU + 绝对位置编码）训练出的结果相差多少？*

`LLaMA`的数据策略策略可以说是一次彻底的“拨乱反正”，从“重模型，轻数据”到“重数据，轻模型”：
1. 数据来源透明：以往模型通常使用大量未公开的“专有”数据，`LLaMA`开创性完全使用了公开可用的数据集；
2. 数据规模和配比：以往模型严重“欠拟合”，模型参数量巨大但训练数据不足（`GPT-3`中，其$1750$亿参数仅用约$3000亿$`Token`训练）；`LLaMA-65B`模型仅用`GPT-3`约$\frac{1}{3}$的参数量，使用了高达$1.4T$`Token`训练；
3. 预处理流程：以往模型清洗方法相对粗略（`GPT-3`主要以来启发式规则和简单对于词频的过滤），`LLaMA`引入了一套业界领先的、系统化的精细过滤流程：
	- 去重：使用`CCNet`流程，在不同数据集和文档内部分别进行全局去重，减少重复记忆；
	- 语言识别：使用`fastText`工具，筛选高质量英文页面；
	- 质量过滤：使用`n-gram`语言模型，基于困惑度等指标过滤低质量内容；
	- 参考源过滤：训练分类器，保留那些更可能成为引用源的页面，提高文本权威性；
	- 针对性清洗：针对不同数据源进行个性化处理，比如清除维基百科的格式代码、`ArXiv`论文的作者信息等。
4. 训练方式：以往做法是让模型在数据上多次迭代，做法简单，但可能导致模型浪费算力“记住”重复内容，甚至过拟合；`LLaMA`采用多数据，少迭代的策略，绝大部分`Token`只经历一轮迭代，仅针对小型高质量数据集，才训练2个`Epoch`，这样做极大限度提高了算力利用率，让模型在尽可能多的独特数据上学习而非重复记忆。
# 结论
`LLaMA`论证了在算力有限的情况下，**极致的数据清洗和充分的训练时长（大`Tokens`）** 相对于**单纯的“堆参数”** 更具备性价比。
