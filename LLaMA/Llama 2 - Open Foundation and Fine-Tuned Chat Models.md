# 背景
`GhatGPT`、`GPT-4`等闭源模型性能领先，开放模型如`LLaMA 1`虽然出现，但是性能仍有明显差距，且`LLaMA 1`仅限于研究使用。
本文的目的在于通过大规模预训练+高质量微调（特别是`RLHF`），使开源模型可以达到甚至超过闭源模型的性能，同时开放免费商用许可。
# `LLaMA 2`

相比于`LLaMA 1`，数据量从$1.4T$提升到$2T$`Token`，并经过清洗和去重，来源同样是公开可用数据，模型内部细节基本没有变化：

|       |                                       |                             |              |
| ----- | ------------------------------------- | --------------------------- | ------------ |
| 组件    | Llama 1                               | Llama 2                     | 说明           |
| 层数    | 7B:32, 13B:40, 33B:60, 65B:80         | 7B:32, 13B:40, 70B:80       | 类似           |
| 隐藏维度  | 7B:4096, 13B:5120, 33B:6656, 65B:8192 | 7B:4096, 13B:5120, 70B:8192 | 70B 与 65B 一致 |
| 注意力头数 | 32                                    | 32                          | MHA          |
| KV 头数 | 32                                    | 7/13B:32, 70B:8             | GQA 用于 70B   |
| 上下文长度 | 2048                                  | 4096                        | 翻倍           |
| 归一化   | RMSNorm (pre-norm)                    | RMSNorm (pre-norm)          | 相同           |
| 激活函数  | SwiGLU                                | SwiGLU                      | 相同           |
| 位置编码  | RoPE                                  | RoPE                        | 相同           |

创新在于`CQA(Grouped-Query Attention)`，用于$70B$模型，`MHA`在长序列推理时，`KV cache`内存占用大，为此，将$32$个查询头分为$8$组，每组共享一个`KV`头（类似`MQA`，但折中），最终效果是推理速度提升约2倍，而性能下降可忽略。
