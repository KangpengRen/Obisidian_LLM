# 背景

在`Mamba`之前，以`S4`为代表的结构化状态空间模型（`SSMs`）虽然通过“`CNN`模式”实现了高效的线性复杂度并行训练，并利用“`RNN模式`”实现了快速自回归推理，解决了长序列建模的效率问题。然而，它在语言建模等任务上却始终无法匹敌`Transformer`。
论文尖锐地指出，这些模型存在一个关键弱点：**缺乏内容感知能力**，无法执行基于内容的推理。
传统`SSMs`的参数是**线性时不变**的，这意味着它们会不加区分地将所有历史信息压缩到状态中。在处理语言时，这种“一视同仁”的压缩方式会导致“选择性失忆”——模型无法像`Transformer`那样，聚焦于关键词，而忽略无意义的虚词）**。
# `Selective SSMs`

为了赋予`SSM`内容感知能力，论文进行了两项关键创新。
## 输入依赖的参数化
这是`Mamba`最核心的贡献。它将SSM中原本固定的参数（`B`, `C`, `∆`）变成了输入的函数，即 `B = s_B(x)`, `C = s_C(x)`, `∆ = s_∆(x)`。
这一改变赋予了模型动态“选择性”的能力。`B`和`C`参数变得**输入依赖**，让模型能根据当前token的内容，决定是否将该`token`的信息写入状态（`B`）或从状态中读出（`C`）。
而`∆`参数通过影响状态更新的步长，充当了“遗忘门”的角色。模型可以学习为重要的token设置大的`∆`（保存重要信息），为不重要的token设置小的`∆`（忽略无关信息）。这样，Mamba的状态就能有选择性地更新，实现高效且内容感知的压缩。
## 重新激活“RNN模式”
然而，`B`, `C`, `∆`变成输入函数后，模型不再是线性时不变的，**无法再使用高效的卷积运算进行并行训练**。为了解决这个矛盾，论文设计了一种**硬件感知的并行算法**。
这个算法充分利用GPU的内存层次结构（HBM和SRAM），采用**内核融合（Kernel Fusion）**、**并行扫描（Parallel Scan）** 等技术，在看似是递归的`RNN模式`下，实现了高效的并行计算。
## 整体架构
Mamba不仅是一个新的SSM层，更是一个完整的神经网络架构。它比`Transformer`更简单，其核心是**均匀堆叠的Mamba块**，每个块都放弃了复杂的MLP（多层感知器）子层，其设计完全围绕`Selective SSM`展开。
一个典型的Mamba块的工作流程如下：
1.  **层归一化**：对输入进行归一化。
2.  **线性投影**：将输入扩展到更大的特征维度。
3.  **卷积**：通过一个短卷积为序列引入局部上下文信息，这是SSM的重要补充。
4.  **激活函数**：通常为`SiLU/Swish`。
5.  **选择性SSM**：如上所述，这是核心的计算单元。
6.  **残差连接**：将块的输入直接加到输出上，促进梯度流动。
# 总结
这也引出了一个更宏大的视角：`Mamba`的贡献不仅是实现了一种更高效的`Transformer`替代品，它其实证明了“选择性”这一能力，并非注意力机制的专利，通过巧妙改造，一个结构更简单的线性循环模型同样能做到。这为未来的序列模型研究，从架构创新转向对“动态计算”更深层次的理解，开辟了全新的可能。
