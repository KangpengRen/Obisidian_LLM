# 背景
在`LSSL`之前，主流序列模型各有各的不足：
- `RNN`：推理高效，但训练慢，难以捕捉长距离依赖；
- `CNN`：训练可并行，但感受野受限，难以建模全局信息；
- `NDE`：理论优雅，但计算成本高，实用性有限。
`LSSL`的核心思想，就是基于控制理论中的线性时不变（`LTI`）系统，构建一个能同时解决以上三个痛点的全能型选手。
# 模型
`LSSL`证明了通过不同的计算方式，可以分别发挥三大模型范式的优势：
- `RNN`模式（推理）：在线性递归模式下逐实践部更新状态，实现恒定的内存和$O(1)$的计算开销；
- `CNN`模式（训练）：在训练时通过展开，转化为一个全局卷积核，利用`FFT`实现并行计算，大幅提升效率；
- `NDE`模式（连续时间）：其微分方程形式使其能够自然地处理连续时间信息，并灵活适应不同的时间采样率。
通俗来说，`RNN`推理快，来一个画面处理一个，`CNN`训练快，可以同时看很多画面，`NDE`可以处理连续时间，`LSSL`并非简单组合以上三种范式，而是从一个更基础的层面——“线性时不变系统（`LTI, Linear Time-Invariant System`）”出发，证明了这三种模型本质都是它的特例。通过控制不同的计算方式，就能分别获得它们的优势。
$$
x^{'}(t) = A x(t) + B u(t)
$$

| 符号      | 含义                       | 通俗比喻                 |
| ------- | ------------------------ | -------------------- |
| `t`     | 连续时间                     | 一个连续流动的时间轴           |
| `x(t)`  | 系统在时刻 `t` 的 **状态**（一个向量） | 你大脑里此刻的 **“记忆/信念”**  |
| `x'(t)` | 状态对时间的 **导数**，即变化率       | 你的记忆 **正在以多快速度改变**   |
| `u(t)`  | 外部 **输入**（也是一个向量）        | 你此刻 **新看到或听到的信息**    |
| `A`     | **状态转移矩阵**               | 你的大脑 **自然遗忘或演化** 的规律 |
| `B`     | **输入矩阵**                 | 你的大脑 **如何吸收新信息** 的规律 |
把这个连续公式变成计算机能用的“离散版本”，作为递归展开：
$$
x_t = Ā * x_{t-1} + B̄ * u_t
$$
这就是一个 **RNN 的更新公式**：
- 新的记忆 `x_t` = 遗忘一部分旧记忆 + 加入新的输入信息
所以 **`x'(t) = A x(t) + B u(t)` 本质上是一个“连续时间的 RNN”**。

作为卷积展开：在`CNN`模式下，LSSL 把状态空间模型展开成一个很长的卷积核，从而可以像 CNN 一样并行训练。
# 总结
`LSSL`确实将`RNN`、`CNN`、`NDE`抽象到一个更底层的数学形式（线性状态空间模型），发现它们在本质上是同一个动力学系统的不同“视角”或“计算模式”。通过把握这个本质，模型可以在训练时选择`CNN`模式（并行卷积）来加速，在推理时选择`RNN`模式（递归更新）来节省内存，在面对非均匀时间戳时选择`NDE`模式（连续微分方程）来灵活适应——真正做到“取长补短”。但理论上可行，工程上跑不动，训练时需要一个超大的中间矩阵，内存爆炸。
