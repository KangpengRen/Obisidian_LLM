# 背景
长上下文建模是大模型的关键能力，但标准注意力的$O(n^2)$复杂度带来了巨大的计算与访存开销。稀疏注意力提供了理论上的效率优势，但此前方案存在三个核心痛点：
- **训练缺失**：大部分稀疏注意力只用于推理阶段，模型主体仍以全注意力模式预训练，导致模型无法在训练中学习最优稀疏模式，能力受损。
- **硬件不匹配**：许多方法虽然在`FLOPs`上减少了，但其内存访问模式不规则，难以充分利用现代GPU的`Tensor Core`和内存带宽。
- **阶段割裂**：多数方法仅优化`Prefill`或`Decoding`之一，难以在全生命周期中保持高效。
此外，稀疏模式难以与GQA等高效架构兼容，也是实际落地中的一大障碍。
# 动态分层稀疏策略
`NSA`的整体框架将输入的$Key$和$Value$组织成时间块，通过三个并行的注意力分支处理，最后通过可学习的门控机制融合输出。

| 分支          | 策略         | 操作方式                   | 作用            |
| :---------- | :--------- | :--------------------- | :------------ |
| **压缩注意力**   | 粗粒度Token压缩 | 将连续Token聚合成块，生成块级摘要    | 捕捉全局轮廓，扩大感受野  |
| **选择注意力**   | 细粒度Token选择 | 对压缩后的块进行轻量评分，挑选关键块     | 保留重要细节        |
| **滑动窗口注意力** | 局部上下文      | 完整保留当前Token附近的窗口内Token | 确保局部精度，防止信息丢失 |
这三个分支各自独立计算注意力模式，绿色区域为实际计算部分，白色区域则跳过不计算，从而实现高效稀疏化。

NSA在算法设计上的两大核心创新：
1. **硬件对齐的系统（Hardware-Aligned System）**
   - 通过**算术强度均衡（Arithmetic Intensity-Balanced）** 的算法设计，最大化Tensor Core利用率和内存访问效率。
   - 以“连续块”为基本操作单元，专门为GPU访存特性设计新的**Triton内核**，实现块状稀疏注意力的高效执行。
1. **训练感知的设计（Training-Aware Design）**
   - 支持端到端原生训练，通过高效的可微分反向传播算子保证训练稳定性。
   - 这使得模型从预训练阶段就以稀疏形式参与计算，能够学习适应稀疏模式，避免能力损失。
