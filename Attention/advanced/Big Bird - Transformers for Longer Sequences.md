# 背景
`Transformer`的全局自注意力机制带来了巨大的$O(n^2)$计算和内存开销，之前的`Reformer`和`Longformer`都试图通过稀疏化来突破这一瓶颈，但有一个理论缺陷：它们设计的稀疏模式，无法保证模型仍具备图灵完备性和通用近似能力。这意味着，之前的模型在追求效率的同时，可能牺牲了理论上处理所有可能性问题的完备性。而`BigBird`的设计目标，正是要实现线性复杂度的同时，严格保证这些重要的理论性质。
# 稀疏注意力模式
`BigBird`的注意力矩阵不是全连接的，而是由三种稀疏模式组合而成：
## 滑动窗口注意力
同`Longformer`
# 全局注意力
同`Longformer`
# 随机注意力
每个`token`再随机关注$r$个其他`token`，可以是全局的，也可以是局部的。随机连接能极大缩短信息传输的“图直径”，使得任意两个`token`之间可以通过少数几步交互，从而增强模型表达能力。
# 组合方式
设序列长度为$n$，窗口大小为$w$，全局节点数为$g$，随机采样数为$r$，总的注意力边数为：
$$
(n \times w) + (n \times g) + (n \times r)
$$
# 总结
`BigBird`的优势在于理论保证和实践性能的平衡。`Longformer`虽然也线性，但缺乏随机连接，在需要全局推理的任务上弱于`BigBird`。随机注意力虽然增强了表达能力，但引入了不确定性，虽然期望上是好的，但可能造成训练不稳定（论文采用随机种子来缓解）。全局节点的选择策略是启发式，不一定最优（后续有人尝试学习全局节点）。
